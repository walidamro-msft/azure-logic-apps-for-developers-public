Logic Apps Automation: Managed Hosting

A managed consumption hosting option in Azure Logic Apps with dedicated compute, VNet integration, stateful-only workflows, and scale-to-zero pricing.

Managed consumptionStateful workflowsDedicated computeScale to zero
Important: Logic Apps Automation supports stateful workflows only. If your solution requires stateless execution, choose a Logic Apps Standard hosting option.

Hosting Position

Important: The hosting name is Logic Apps Automation. Treat it as its own managed consumption option and do not assume Workflow Service Plan limits or pricing apply.

Logic Apps Automation is positioned as a managed consumption service with dedicated compute, VNet integration, and scale-to-zero behavior. It is distinct from the regular shared Consumption hosting model.

Why This Hosting Plan Is Needed

Logic Apps Automation addresses workloads that need more isolation and networking control than regular shared Consumption, but do not need an always-running Workflow Service Plan. It combines dedicated, Azure-managed compute with VNet integration and the ability to scale to zero when no work is running.

Reduce idle compute

Intermittent automation can avoid keeping reserved Workflow Service Plan instances active throughout quiet periods.

Keep dedicated execution

Projects can use dedicated compute instead of sharing the regular Consumption runtime.

Use stateful-only hosting

This option is designed for stateful workflows and should be selected only when stateful behavior is required.

Reach private systems

VNet integration supports workflows that must call private APIs, databases, and line-of-business services.

Limit platform ownership

Azure manages the compute environment, avoiding the Kubernetes and SQL Server responsibilities required by Hybrid hosting.

Pricing Availability

Logic Apps Automation uses a base + per vCPU-second with scale-to-zero billing shape. Model both the fixed base cost and active compute time.

Estimated charge shape = published base charge + (active vCPU-seconds × published vCPU-second rate) + dependent services

Do not substitute Workflow Standard, Hybrid, or regular Consumption rates. Obtain the applicable base and vCPU-second rates for your subscription and region, then apply them to measured active vCPU-seconds.

Evaluation Checklist

Availability

Verify that Logic Apps Automation can be created in the target subscription and region.

Pricing

Capture the portal's base and vCPU-second rates and confirm how scale-to-zero periods are billed.

Workflow model

Confirm that all required flows are stateful and align with the stateful-only hosting scope.

Feature parity

Test required triggers, actions, deployment tooling, limits, and monitoring.

Networking

Validate VNet integration, DNS, private endpoints, outbound routing, and connector dependencies.

Support

Review support terms, SLA status, upgrade path, and production-use policy.

Fallback

Keep an alternate hosting design available if availability, capabilities, or support terms do not meet the workload requirement.

Projects That Are a Good Fit

Scheduled business automation

Nightly reconciliation, periodic report generation, invoice processing, and scheduled data synchronization with long idle periods.

Bursty event processing

Projects that process occasional batches or event bursts and need dedicated capacity only while work is active.

Private application integration

Workflows that connect cloud services to private APIs, databases, or enterprise systems through a VNet.

Departmental automation

Groups of related operational workflows that benefit from dedicated compute and independent scaling.

Modernized enterprise processes

Integration projects moving from scripts or scheduled middleware jobs that need managed orchestration without an always-on host.

Stateful integration workflows

Workloads that require persisted state and benefit from dedicated managed consumption compute.
