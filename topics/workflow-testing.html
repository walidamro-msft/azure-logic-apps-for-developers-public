Workflow Testing

Use mock outputs (static results) to test workflow branches, responses, and error handling without calling production services.

Consumption + StandardSafe validation

What Static Results Do

Static results let you return test outputs from an action so the action doesn't call the target system. This approach helps you validate control flow and data handling while protecting live dependencies.

  • Simulate success and failure paths on demand.
  • Test workflow expressions and branch conditions with known payloads.
  • Exercise retry and error scopes without generating real downstream effects.
  • Validate response shaping before integration cutover.

Prerequisites and Limits

  • Have an Azure subscription and an existing logic app workflow.
  • Select the action you want to test in the designer.
  • Static results apply to actions, not triggers.
  • Enable and disable static results manually per action.

Important

An action with static results enabled shows a test beaker icon in the designer and run details.

Set Up Mock Outputs on an Action

  1. Open your workflow in the Azure portal designer.
  2. Select the target action, then open the Testing tab.
  3. Select Enable Static Result.
  4. Pick the response fields you want to mock.
  5. Set values for status, output, or error based on your test case.
  6. Save and run the workflow to validate behavior.

Field Selection Guide (HTTP Action Example)

StatusAlwaysChoose Succeeded or Failed to drive branch logic.
OutputsWhen status is SucceededReturn status code, headers, and test body payload.
ErrorWhen status is FailedReturn test error message and optional error code.
CodeOptionalInject specific code values to test condition checks and handling.

Validate and Inspect Test Runs

  • Run the workflow and inspect the action output and downstream branches.
  • For Consumption workflows, use run history and check the Static Results column.
  • Open run details and confirm which actions used static results.
  • Compare outputs against expected payload contracts before enabling live integration.

Disable Static Results Safely

  1. Select the action in the designer.
  2. Open the Testing tab.
  3. Select Disable Static Result and save.

Turning the setting off does not erase your previously configured mock values, so you can re-enable and reuse them later.

Testing Best Practices

  • Keep a small set of named test scenarios for happy path, transient fault, and hard failure.
  • Mock at integration boundaries where side effects are expensive or risky.
  • Avoid leaving static results enabled after test validation in shared environments.
  • Pair static-result tests with at least one live-endpoint verification before release.
