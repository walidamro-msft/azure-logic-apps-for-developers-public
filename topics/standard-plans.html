Azure Logic Apps Standard Hosting

One single-tenant runtime model with three deployment choices: managed Workflow Service Plan, fully isolated App Service Environment v3, or customer-managed Hybrid infrastructure.

Workflow Standard Tiers

WS1, WS2, and WS3 are fixed vCPU-and-memory sizes that apply only to Workflow Service Plan. Compute is billed while each plan instance is enabled, and scale-out adds another complete tier instance.

Pricing reference: See Workflow Service Plan pricing for tier capacities, the current East US teaching example, and the monthly calculation.

Stateful and Stateless Workflows

CharacteristicStatefulStateless
PersistenceInputs, outputs, state, and run history stored externallyState remains in memory; run history is off by default
ReliabilityInterrupted runs can resume from persisted stateCaller must resubmit after interruption
DurationLong-running workflows, 90-day defaultShort synchronous workflows, typically under 5 minutes
PerformanceDurable orchestration with storage overheadLower latency and higher throughput for small payloads
Connector triggersBuilt-in and managed connector triggersPush triggers; managed polling triggers are not supported
Best payload fitLarger or durable messagesSmall messages, ideally under 64 KB total

Workflow type is selected at creation. Changing a workflow between stateful and stateless afterward causes runtime errors.

Built-in and Shared Connector Billing

Standard compute includes unlimited built-in operation executions. Built-in connectors run in the same process as the workflow runtime, often improving latency and throughput.

Connector typeExecution locationBilling
Built-inIn-process on the single-tenant runtimeIncluded in Standard compute
Custom built-inIn-process extensionIncluded in Standard compute
Shared Standard managed connectorShared global connector infrastructure$0.000125 per call in the East US sample
Shared Enterprise managed connectorShared global connector infrastructure$0.001 per call in the East US sample

Unlike Consumption, Standard meters paid managed connector operations per call. Pagination or chunking can therefore produce multiple billable calls. Prefer an appropriate built-in connector when feature parity, authentication, and operational requirements allow.

Networking, Isolation, and Data Residency

VNet integration

Reach private backends directly from the single-tenant runtime and control outbound routing.

Private endpoints

Expose the logic app through private inbound connectivity instead of a public endpoint.

Regional data

Workflow data stays in the region where the Standard logic app is deployed.

ASE v3

Use Windows Isolated v2 App Service Plans when fully isolated dedicated hosting is required.

Shared managed connectors still run outside the single-tenant process. Include their FQDNs, service tags, and connection behavior in strict firewall designs.

Development and Deployment Model

A Standard logic app is an application project containing the runtime configuration, workflows, connections, and artifacts. You can run and debug locally with the Azure Logic Apps (Standard) VS Code extension, package the project, and deploy application artifacts separately from infrastructure.

Storage and Additional Cost

Stateful Standard workflows use your Azure Storage account for runtime state, scheduling, and run history. Storage capacity and transactions are billed separately under Azure Storage pricing. Shared managed connectors, integration accounts, networking, and dependent services are also outside the WS compute estimate.

Cost check: compare total Standard compute + instance count + storage + shared connectors against Consumption operation volume. Compute alone is not the complete comparison.

When Standard Fits

Strong fit

Private networking, several related workflows, sustained throughput, built-in connector volume, local development, and regional data residency.

Capacity warning

Reserved compute is paid while enabled. Load test CPU, memory, storage transactions, trigger latency, and scale-out behavior before selecting WS1, WS2, or WS3.

Connector warning

Do not assume all operations are included. Shared managed connector calls remain separately metered even on Standard.
