Managed Standard Architecture

Azure runs the single-tenant Logic Apps runtime on a Workflow Service Plan. One plan can host Standard logic app resources, and each resource can package multiple stateful and stateless workflows. Scale-out adds plan instances.

East US Pricing

The public East US meters are $0.1997 per vCPU-hour and $0.0143 per GiB-hour. These examples assume one enabled instance for 730 hours.

Capacity and Scaling

Start with measurements

Load test CPU, memory, trigger latency, storage transactions, and connector throttling before selecting a production tier.

Scale up

Move from WS1 to WS2 or WS3 when a workflow app needs more CPU and memory per instance.

Scale out

Add instances for concurrency and resilience. Each active instance adds the complete tier cost.

Share deliberately

Multiple apps can improve plan utilization, but noisy-neighbor behavior remains your capacity-management responsibility.

When to Choose It

Choose Workflow Service Plan for the normal managed Standard experience: private access to Azure resources, several related workflows, predictable capacity, local development, and regional workflow data without managing an isolated App Service environment or Kubernetes platform.

Choose ASE v3 for fully isolated App Service infrastructure, or Hybrid when execution must run on customer-managed Kubernetes.
