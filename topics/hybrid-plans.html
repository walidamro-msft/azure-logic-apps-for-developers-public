Logic Apps Standard Hybrid Hosting

Run the Standard runtime on customer-managed Azure Arc-enabled Kubernetes for on-premises, edge, and partially connected integration.

Standard runtimeAzure ArcKubernetesPer vCPU-hour

Customer-Managed Runtime

Hybrid hosting deploys the Logic Apps Standard runtime through the Azure Container Apps extension on Azure Arc-enabled Kubernetes. Workflow execution and data can remain close to local systems while Azure provides the control-plane experience.

    AZ[Azure control plane] --> ARC[Azure Arc-enabled Kubernetes]

East US Pricing

The public Logic Apps Hybrid meter is $0.18 per allocated vCPU-hour in East US. Configure 0.25 to 2 vCPUs per replica; each replica is limited to 2 vCPUs.

Logic Apps charge = allocated vCPU per replica × replica count × active hours × $0.18

Additional Costs

The Hybrid vCPU meter is only the Logic Apps software charge. Add the Kubernetes nodes and supporting infrastructure, SQL Server infrastructure and licensing, storage, networking, monitoring, backup, and operations. Shared managed connector calls remain separately metered.

Cost comparison: compare the complete customer-managed platform cost with Workflow Service Plan or ASE v3, not only the $0.18 Logic Apps vCPU rate.

When to Choose It

Strong fit

Data processing must stay on premises, latency to local systems is critical, or cloud connectivity is constrained.

Platform fit

Your organization already operates Kubernetes, Azure Arc, SQL Server, and the required reliability and security controls.

Weak fit

You want Azure to manage compute and platform availability. Workflow Service Plan is usually simpler.

Capacity warning

Replica counts and vCPU allocations affect both Logic Apps charges and the cluster capacity you must supply.
