Portal Menus

Menu-by-menu guide for the left navigation under a Logic App (Standard) resource running on Workflow Service Plan in Azure portal.

Workflow Service PlanAzure PortalOperations

How to Read This Guide

  • Some menu items are Logic Apps-specific, while others are inherited from Azure App Service, Azure Monitor, or Azure Resource Manager.
  • The exact menu can vary by region, feature flags, and role permissions.
  • Preview items can change behavior or naming before general availability.
Important: A Logic App (Standard) resource is hosted on App Service infrastructure, so many platform pages map to App Service guidance even though you are working with workflows.

Agents > MCP serversManage Model Context Protocol server endpoints shown for agent scenarios.Portal feature surface; preview-oriented.
Agents > Knowledge baseManage knowledge resources exposed for agent-assisted scenarios.Portal feature surface; preview-oriented.
Assembly type in Add AssemblyWhen to choose it
Client/SDK Assembly (.NET Framework)Reference supported .NET Framework client SDK libraries needed by your workflow operations.
Client/SDK Assembly (Java)Reference supported Java SDK libraries for Java-based assembly scenarios.
Custom Assembly (.NET Framework)Use your own .NET Framework assembly built for custom workflow logic integration.
Custom Assembly (.NET 8)Use your own assembly built for .NET 8 runtime compatibility.
Custom Assembly (.NET 10)Use your own assembly built for .NET 10 runtime compatibility where your workload and dependencies require it.
Important: Match the assembly type to your target runtime and dependency chain. A version mismatch can cause load failures at design time or run time.
Note: Agent-related pages are newer portal capabilities and can differ by tenant rollout, region, and enabled features.
