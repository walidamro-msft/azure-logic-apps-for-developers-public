Standard on App Service Environment v3

Run Logic Apps Standard on dedicated, fully isolated App Service infrastructure using Windows Isolated v2 plans.

Standard runtimeFully isolatedWindows onlyIsolated v2

Dedicated Environment

The Standard runtime runs inside your App Service Environment v3. Logic apps inherit the ASE network boundary, routing, inbound model, and operational controls. Logic Apps Standard on ASE v3 supports Windows plans only.

East US Pricing

ASE-hosted Logic Apps are billed through the Windows Isolated v2 App Service Plan, not the Workflow Standard vCPU and memory meters. These public pay-as-you-go East US samples assume one instance for 730 hours.

Billing boundary: for Logic Apps Standard on ASE v3, Microsoft documents the charge as the Isolated v2 App Service Plan instances used to run the workflows. There is no additional Logic Apps execution charge for built-in operations.

Complete Cost Boundary

  • Built-in workflow operations are included in the App Service Plan capacity.
  • Shared managed connector calls remain separately billed at Logic Apps connector rates.
  • Stateful workflows use an Azure Storage account billed separately.
  • Private endpoints, gateways, firewalls, DNS, monitoring, and dependent services can add costs.
  • Reserved pricing can differ from the pay-as-you-go samples shown here.

Networking and Operations

Inherited isolation

Workflow apps use the ASE's dedicated environment and VNet topology rather than the shared Workflow Service Plan infrastructure.

Inbound design

Choose internal or external access patterns at the ASE level and coordinate private DNS and certificates.

Outbound design

Apply route tables, firewalls, and network security controls while preserving required Azure service and connector access.

Shared platform

Multiple compatible App Service workloads can share ASE capacity, improving utilization but expanding the operational scope.

When to Choose It

Choose ASE v3 when policy requires fully isolated App Service infrastructure, high scale in a dedicated environment, or a shared isolated platform for compatible App Service workloads. Use Workflow Service Plan when private networking is sufficient and you do not need to operate an ASE.

Do not confuse the meters: legacy Logic Apps Integration Service Environment pricing is unrelated to Standard on App Service Environment v3.
