Azure Logic Apps Developer Workshop

You are invited to a hands-on Azure Logic Apps workshop focused on designing, building, and operating automated workflows and integration solutions.

What we will cover:
- Azure Logic Apps fundamentals: workflows, triggers, actions, connectors, and control flow
- Logic App structure and deployment boundaries across Consumption and Standard
- Runtime lifecycle: trigger dispatch, workflow execution, state management, scaling, and billing
- Hosting plan selection: Consumption versus Standard tradeoffs for performance, cost, networking, and operations
- Advanced integration patterns using conditions, loops, scopes, APIs, Service Bus, and Azure Functions
- Security and governance: managed identities, RBAC, secrets, network access, and policy controls
- Monitoring and troubleshooting with run history, metrics, logs, and Application Insights
- Workflow design, testing, deployment, and CI/CD practices
- Building and managing workflows with the Azure portal and Visual Studio Code

Who should attend:
- Cloud architects, integration developers, application developers, platform engineers, and DevOps engineers

Prerequisites:
- Basic Azure knowledge
- Familiarity with APIs, events, or workflow automation concepts
- Azure subscription access for hands-on labs (recommended)

Outcome:
By the end of this workshop, attendees will be able to choose the right Logic Apps hosting model, design secure and observable workflows, and build and deploy production-ready integration solutions.

Recommended Delivery Flow

  1. Open the workshop home page and explain the module structure.
  2. Spend most of the time on Overview, Logic Apps Structure, Runtime Lifecycle, Core Concepts, Hosting Plans, and Build & Manage.
  3. Use Advanced Patterns, Security, and Design Guidance as depth modules based on audience maturity.
  4. Close with the Use Cases page so attendees leave with applied scenarios, not just platform features.
