How this workshop runs

Move from the platform and runtime model into workflow design, hosting decisions, production readiness, testing, applied scenarios, and repeatable delivery.

Introduce the platform

Define Logic Apps, why teams use it, and where it fits in Azure integration architecture. Overview

Clarify deployment boundaries

Show how workflows map to Logic App resources across Consumption and Standard hosting. Logic Apps Structure

Open the runtime box

Explain workflow loading, trigger dispatch, job orchestration, state storage, connectors, scaling, and billing. Runtime Lifecycle

Teach the workflow model

Triggers, actions, connectors, and stateful versus stateless workflows. Core Concepts

Move into advanced patterns

Conditions, loops, scopes, APIs, hybrid access, and extension with Functions and Service Bus. Advanced Patterns

Choose the right hosting option

Use Standard versus Consumption tradeoffs around performance, cost, networking, and workflow type. Hosting Plans

Design for production

Apply maintainability, concurrency, throughput, networking, and architecture guidance before implementation hardens. Design Guidance

Select connectors and B2B capabilities

Compare built-in and managed connector runtime, cost, and availability, then decide whether the solution needs an Integration Account. Connectors and Integration Accounts

Secure and govern the solution

Choose identities, protect secrets and endpoints, apply least privilege, and design within current platform limits. Security & Limits

Operate and recover safely

Use run history, Azure Monitor, diagnostic logs, alerts, correlation, and idempotent recovery practices. Monitoring & Troubleshooting

Validate workflow behavior

Test branches, failures, and payload handling with static results and controlled outputs before calling live systems. Workflow Testing

Apply the platform to real scenarios

Connect the design choices to business automation, integration, event handling, scheduled processing, and governed agent flows. Use Cases and Agentic Workflows

Build, deploy, and manage

Author in the portal or VS Code, then package, parameterize, validate, and promote workflows through controlled environments. Build & Manage
