Lab 04: Event Grid Blob Processor

React to new blobs without polling and create a compact audit record for each incoming file.

Azure portalConsumption~45 min

Scenario and Objectives

A document intake process needs immediate notification when a file arrives. The workflow records event metadata in a separate container for later audit and downstream processing.

  • Create an Event Grid-triggered workflow.
  • Filter events to one source container.
  • Extract event metadata and write an audit artifact.
  • Trace the event from storage through run history.

Prepare Resources

  1. Create a general-purpose v2 storage account with Event Grid support.
  2. Create private blob containers named incoming and audit.
  3. Create a Logic App (Consumption) in the same resource group.
  4. Enable its system-assigned identity and grant Storage Blob Data Contributor only where needed if your Blob connector operation supports managed identity.

Build the Event Workflow

  1. In the designer, select the Event Grid trigger When a resource event occurs.
  2. Select the subscription, resource type Microsoft.Storage.StorageAccounts, and your storage account resource.
  3. Select the Blob Created event type.
  4. Add an advanced subject filter that begins with /blobServices/default/containers/incoming/. Save once so Azure can create the event subscription.
  5. Add a Compose action named Audit record and build the JSON object below. Use trigger dynamic content for the event fields where available.
  6. Add Azure Blob Storage Create blob, target the audit container, and use the filename and content expressions below.
  7. Save the workflow.

Audit record

{
  "eventId": "@{triggerBody()?['id']}",
  "eventType": "@{triggerBody()?['eventType']}",
  "subject": "@{triggerBody()?['subject']}",
  "blobUrl": "@{triggerBody()?['data']?['url']}",
  "receivedUtc": "@{utcNow()}",
  "workflowRunId": "@{workflow()?['run']?['name']}"
}
File name: @concat('event-', triggerBody()?['id'], '.json')
Blob content: @string(outputs('Audit_record'))

Connector output shape

If your trigger exposes the Event Grid event as an array or uses different dynamic-content tokens, select the values from the designer and inspect the first trigger output in run history. Do not guess property nesting in production workflows.

Test the Event

  1. Upload a small text file named order-1001.txt to incoming.
  2. Wait for the workflow run, then open run history and inspect the trigger body.
  3. Open the audit container and inspect the generated JSON blob.
  4. Upload a file directly to audit and confirm that no new workflow run starts.
  • The audit record identifies the source blob URL and event ID.
  • Exactly one run occurs for the upload to incoming.
  • No loop occurs when the workflow creates the audit blob.

Troubleshooting

SymptomCheck
No run startsOpen the storage account Event Grid subscription and check provisioning status, event type, and subject prefix.
Repeated runs occurNarrow the subject prefix to incoming and verify the output action writes only to audit.
Audit action is unauthorizedConfirm connector authentication and the identity's data-plane role assignment.

Cleanup and References

Delete the Event Grid subscription before deleting the logic app, then delete storage data and the resource group when no longer needed.
