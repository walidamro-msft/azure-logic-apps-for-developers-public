Lab 02: Timer Daily Report

Use a recurrence trigger to create a timestamped daily sales summary and store it as a blob.

Azure portalConsumption~40 min

Scenario and Objectives

An operations team needs a report generated every morning. You will use fixed lab data to make the first run repeatable, then isolate the point where a production data source would connect.

  • Configure a timezone-aware recurrence.
  • Transform an array into a summary and CSV table.
  • Create a dated report in Azure Blob Storage.
  • Run the schedule on demand and inspect history.

Prepare Storage

  1. Create a general-purpose v2 storage account in the same region as the logic app.
  2. Under Data storage > Containers, create a private container named reports.
  3. Create a Consumption logic app named la-daily-report-<suffix>.

Authentication

When your selected Blob connector operation supports managed identity, enable the logic app identity, assign only Storage Blob Data Contributor on the target storage scope, and choose managed identity for the connection. Otherwise, use the lab connection only temporarily and remove it during cleanup.

Build the Workflow

  1. Choose the Recurrence trigger. Set Interval = 1, Frequency = Day, and select your timezone. If you set Start time, use ISO 8601 format (for example, 2026-07-12T10:00:00 with timezone selected, or 2026-07-12T16:00:00Z in UTC). For this lab, you can also leave Start time empty and run manually. Leave At these hours and At these minutes empty.
  2. Add a Compose action and set the title to Sales Data (or Sales data), then paste the sample array in Inputs.
  3. Add Create CSV table. Set From to the Outputs token from Sales Data (preferred). If you manually paste the same JSON array into From, it also works for this lab, but using the action output avoids duplication.
  4. Add Azure Blob Storage Create blob (V2). In the connection dialog, use Managed identity when available and already authorized. If managed identity is not available in your lab path, choose Access key as the temporary lab fallback. Do not pick Service principal authentication unless you already have a client ID, secret, and tenant prepared.
  5. In Create blob (V2), set Folder path to the reports container, then set Blob name and Blob content using the values below.
  6. Save, then select Run trigger > Run so you do not have to wait for the scheduled time.

Sample sales data

[
  { "region": "North", "orders": 12, "revenue": 1450.25 },
  { "region": "South", "orders": 9, "revenue": 980.50 },
  { "region": "West", "orders": 15, "revenue": 1765.00 }
]

Blob values

Folder path: /reports
    Blob name (file name): @concat('daily-sales-', formatDateTime(utcNow(), 'yyyy-MM-dd-HHmmss'), '.csv')
    Blob content: @body('Create_CSV_table')
Important: reports is the container/folder path, not the blob file name. In Create blob (V2), set Folder path to /reports (or pick it with the folder icon), then put the timestamp expression in Blob name.
Note: the timestamp includes seconds so repeated manual tests create separate files. A production daily workflow might instead use yyyy-MM-dd and an explicit overwrite policy.

Validate the Report

  • The manual run succeeds from Recurrence through Create blob.
  • The reports container contains a new timestamped CSV file.
  • The downloaded CSV contains three data rows and the expected headers.
  • The trigger configuration shows the intended timezone and future schedule.

Open the run and compare the Compose output, CSV action output, and blob action inputs. This is the fastest way to locate a malformed transformation.

Extend the Design

Replace the fixed Compose action with a data-source action such as a database query, HTTP API call, or storage read. Keep the transformation and delivery actions downstream so you can test each boundary independently.

  • Add a condition that skips blob creation when the input array is empty.
  • Add a failure scope configured with Run after to capture connector errors.
  • Send a completion notification containing the blob path, not a storage key.

Troubleshooting

SymptomCheck
Blob action is unauthorizedConfirm the connection identity and data-plane role assignment; RBAC propagation can take several minutes.
Managed identity used with this operation no longer exists / Invalid connectionSelect Change connection and create a new connection. For this lab, use Access Key as a quick fallback, or re-enable the logic app identity and reassign Storage Blob Data Contributor before reconnecting with managed identity.
CSV contains one JSON cellSet Create CSV table From to the Compose array output, not its string representation.
Schedule runs at the wrong timeSet the trigger timezone and start time explicitly; inspect trigger history for the evaluated schedule.
